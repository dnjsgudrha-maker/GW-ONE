GW ONE v6.0 운영 안정화 버전

[핵심 변경]
1. 기사·본사 6:4 및 비율 지정 기능 완전 제거
2. 기사가 작업을 등록해도 문서에는 담당기사 이름만 표시
3. 상호, 대표자, 사업자등록번호, 연락처, 이메일, 주소, 직인은 본사 정보만 사용
4. 기존 홈페이지 + GW ONE 통합 Firestore 규칙 유지
5. 기존 작업을 수정 저장하면 과거 비율 정산 필드도 자동 삭제

[본사 정보 적용]
- 최고관리자가 내 업체정보에서 본사 정보를 저장하면 승인된 사용자 프로필로 동기화됩니다.
- 신규 기사 승인 시에도 본사 정보와 본사 직인이 함께 저장됩니다.

[배포]
1. npm install
2. npm run build
3. firebase deploy --only hosting

[Firestore 규칙]
압축파일 안 firestore.rules는 홈페이지와 GW ONE 통합 규칙입니다.
현재 운영 중인 통합 규칙과 동일한 구조이므로 홈페이지 전용 규칙으로 덮어쓰지 마세요.
